For a page `/pages/film.page.js`, a Route String can be defined in a `/pages/film.page.route.js` adjacent file.

```js
// /pages/film.page.route.js

// Match URLs `/film/1`, `/film/2`, ...
export default '/film/:filmId'
```

If the URL matches, the value of `filmId` is available at `pageContext.routeParams.filmId`.

We can implement [catch-all routes](/catch-all).

The precedence of Route Strings is showcased at [`precedence.spec.ts`](https://github.com/brillout/vite-plugin-ssr/blob/master/vite-plugin-ssr/shared/route/precedence.spec.ts).

For more advanced routing, we can use [Route Functions](/route-function) instead of Route Strings.

`vite-plugin-ssr`'s Route String implementation is based on [React Router's `matchPath()`](https://reactrouter.com/web/api/matchPath).
