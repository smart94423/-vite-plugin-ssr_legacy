For a page `/pages/film.page.js`, a Route String can be defined in a `/pages/film.page.route.js` adjacent file.

```js
// /pages/film.page.route.js

// Match URLs `/film/1`, `/film/2`, ...
export default '/film/:filmId'
```

The value of `filmId` is available at `pageContext.routeParams.filmId`.

## Multiple Parameters

We can define multiple parameters.

```js
// /pages/film.page.route.js

// Parameters available at:
//  - `pageContext.routeParams.year`
//  - `pageContext.routeParams.slug`
export default '/news/:year/:slug'
```

## Catch-All

We can use Route Strings to implement [catch-all routes](/catch-all).

## Precedence

The precedence of Route Strings is determined by specificity, for example:
- Most specific `/about/team` first (matches only a single URL)
- Less specific `/about/:path` second (also matches `/about/company`, `/about/vision`, ...)
- Less specific `/about/*` third (also matches `/about/also/nested/path`)
- Least specific `/*` last (matches all URLs)

Further examples at [`precedence.spec.ts`](https://github.com/brillout/vite-plugin-ssr/blob/master/vite-plugin-ssr/shared/route/precedence.spec.ts).
