```ts
// Redirect the user to another URL
redirect(url: `/${string}` | `https://${string}` | `http://${string}`)
```

```js
import { redirect } from 'vite-plugin-ssr/abort'

function onSomeHook() {
  if (someCondition) {
    throw redirect('/some-url')
  }
}
```

> By default, `throw redirect()` makes a temporary redirection (using the HTTP status code `302`). We expect that permanent redirections aren't need for programmatically defined redirections. However, if you do, let us know (there is a hidden option to change the status code).

While it's most commonly used with [`guard()`](/guard) or [`onBeforeRender()`](/onBeforeRender), you can use it with any hook.

Common use cases:

- Authentication and authorization, see <Link href="/auth#login-flow" doNotInferSectionTitle={true} />.
- Data fetching error handling, see <Link href="/data-fetching#error-handling" doNotInferSectionTitle={true} />.

If `throw redirect()` doesn't work, then see <Link href="/abort#how-to-debug" doNotInferSectionTitle={true} noBreadcrumb={true} />.
