The <Link text="V1 design" href="/migration/v1-design" /> adds new foundational capabilities. For example, it enables you to create <Link text="custom configs" href="/meta" /> such as `ssr: boolean`.

```js
// /pages/admin/+config.h.js

export default {
  // Render the Admin Panel as SPA
  ssr: false
}
```

> Vite-plugin-ssr will soon release <Link text="built-in renderers" href="/renderers" /> for React/Vue/Solid which already implement such `ssr` toggle. (Except for React since RSC supersedes such `ssr` toggle.)

You'll even be able to choose and switch renderers on a page-by-page basis:

```js
// /pages/admin/+config.h.js

import graphqlRenderer from 'vike-react-relay'

export default {
  // Use GraphQL for the Admin Panel
  ...graphqlRenderer
}
```

> Vite-plugin-ssr is going to be [renamed to Vike](https://github.com/brillout/vite-plugin-ssr/issues/736).

```js
// /pages/product/@id/+config.h.js

import rpc from 'vike-react-telefunc'

export default {
  // Use React + RPC for the product pages
  ...rpc
}
```

```js
// /pages/live-ticker/+config.h.js

import solid from 'vike-solid'

export default {
  // Use Solid for this page while the rest of our app uses React
  ...solid
}
```

This is only the tip of the iceberg. The V1 design also brings new capabilities for buidling meta frameworks on top of vite-plugin-ssr.
