The <Link text="V1 design" href="/migration/v1-design" /> adds new foundational capabilities.

You'll be able to implement <Link text="custom configs" href="/meta" />, for example `ssr: boolean` to seamlessly toggle render modes:

```js
// /pages/admin/+config.js

export default {
  // Render the Admin Panel as SPA
  ssr: false
}
```

> `+config.js` is explained at:
>  - <Link href="/architecture#config-js" />
>  - <Link href="/config" />

> VPS will release official React/Vue/Solid renderers that already implement `config.ssr`. (You'll be able to eject these renderers for complete control.)

You'll even be able to toggle renderers:

```js
// /pages/admin/+config.js

import graphqlRenderer from 'vike-react-relay'

export default {
  // Use GraphQL for the Admin Panel
  ...graphqlRenderer
}
```

> The `vite-plugin-ssr` project is going to be [renamed to Vike](https://github.com/brillout/vite-plugin-ssr/issues/736).

```js
// /pages/product/@id/+config.js

import rpcRenderer from 'vike-react-telefunc'

export default {
  // Use RPC for the product pages
  ...rpcRenderer
}
```

```js
// /pages/live-ticker/+config.js

import solidRenderer from 'vike-solid'

export default {
  // Use Solid for this page while the rest of our app uses React
  ...solidRenderer
}
```

This is only the tip of the iceberg. The V1 design also brings new foundational capabilities for buidling frameworks on top of VPS.
