Configurations defined by <Link text={<code>+config.js</code>} href="/config" />.

```ts
// +config.ts

// The list is also available over TypeScript
import type { Config } from 'vite-plugin-ssr/types'

export default {
  // ...
} satisfies Config
```

## Configs

### Basics

- [**`Page`**](/Page): <TextType>unknown</TextType> The root UI component of the page.
- [**`route`**](/routing): <TextType>string | Function</TextType> The page's route.
- [**`prerender`**](/prerender-config): Whether to <Link text="pre-render" href="/preredering" /> the page, and pre-rendering settings.
- [**`client`**](/client): <TextType>string</TextType> Additional client entry.
- [**`prefetchStaticAssets`**](/clientRouting#link-prefetching): Link prefetching settings.
- [**`iKnowThePerformanceRisksOfAsyncRouteFunctions`**](/route-function#async): <TextType>boolean</TextType> Enable async Route Functions.

### Advanced

> Usually only used by <Link href="/vike-packages" noBreadcrumb />. Most users don't need to know about these.

- [**`meta`**](/meta): Create new or modify existing configurations.
- [**`passToClient`**](/passToClient): <TextType>string[]</TextType> Determines what `pageContext` properties are sent to the client-side.
- [**`clientRouting`**](/clientRouting): <TextType>boolean</TextType> Enable Client Routing.
- [**`hydrationCanBeAborted`**](/clientRouting#usage-options): <TextType>boolean</TextType>  Whether the <Link text="UI framework" href="/ui-framework" /> allows hydrations to be aborted.

## Hooks

### Basics

- [**`onHydrationEnd`**](/clientRouting#usage-options): <TextEnv>client-side</TextEnv> Called after the page is hydrated.
- [**`onPageTransitionStart`**](/clientRouting#usage-options): <TextEnv>client-side</TextEnv> Called before the user navigates to a new page.
- [**`onPageTransitionEnd`**](/clientRouting#usage-options): <TextEnv>client-side</TextEnv> Called after the user navigates to a new page.
- [**`onPrerenderStart`**](/onPrerenderStart): <TextEnv>server-side</TextEnv> Page-specific hook called when pre-rendering starts.
- [**`onBeforePrerenderStart`**](/onBeforePrerenderStart): <TextEnv>server-side</TextEnv> Global hook called before the whole pre-rendering process starts.

### Advanced

> Usually only used by <Link href="/vike-packages" noBreadcrumb />. Most users don't need to know about these.

- [**`onBeforeRender`**](/onBeforeRender): <TextEnv>server-side (and potentially client-side)</TextEnv> Called before the page is rendered, usually for fetching data.
- [**`onRenderHtml`**](/onRenderHtml): <TextEnv>server-side</TextEnv> Called when page is rendered to HTML on the server-side.
- [**`onRenderClient`**](/onRenderClient): <TextEnv>client-side</TextEnv> Called when page is rendered on the client-side.
- [**`onBeforeRoute`**](/onBeforeRoute): <TextEnv>server (and potentially client-side)</TextEnv> Called before the URL is routed to a page.

## Custom

New configurations can be created by using <Link text="meta" href="/meta" />.
